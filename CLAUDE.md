# LP ファーストビュー制作ガイドライン

このプロジェクトは、日本市場向けLPのファーストビュー（ヒーローセクション）を高品質に制作するためのテンプレートです。

## 制作フロー

1. `docs/brief.md` をコピーして案件情報を埋める
2. `docs/hero-patterns.md` からレイアウトパターンを選ぶ
3. `docs/design-system.md` でトーン・配色を決める
4. `docs/copy-structure.md` でコピーの型を確認する
5. コーディングに入る

## 日本のLP制作 鉄則ルール

### 1. 情報密度は「高く」が基本
- 海外LPのようなミニマル・余白たっぷりは日本のCVR型LPでは逆効果
- ファーストビューに「何のサービスか」「誰向けか」「なぜ選ぶべきか」を全部入れる
- 要素が多い＝悪ではない。整理されていれば情報量は武器になる

### 2. フォントサイズの強弱を極端につける
- メインコピーは画面幅の50〜70%を占めるくらい大きく
- サブコピーとの差を3倍以上つける（例：48px vs 14px）
- 1つのコピー内でも文字サイズ・色・太さを変えて視線誘導する
- 例：「完済**まで**」の「まで」を別サイズにする等

### 3. 文字の装飾パターンを使い分ける
- 蛍光マーカー風の下線（黄色・緑）
- 文字色の部分変更（キーワードだけアクセントカラー）
- 吹き出し・バッジ・ラベル風の囲み
- 縁取り文字（text-stroke）で写真の上でも読ませる
- ドットの下線、波線の下線

### 4. 写真は「人物」が最強
- 信頼感：スーツの専門家（腕組み・正面向き）
- 親しみ：笑顔のスタッフ・利用者
- 憧れ：タレント・モデル
- 写真は画面の40〜60%を占めるべき
- Unsplashの画像を使う場合、日本人の写真を優先する

### 5. 信頼バッジは必須
- ファーストビュー下部に3〜4個並べる
- アイコン + 短いテキスト（「相談料0円」「実績1000件」等）
- 角丸の囲みまたはカード形式で視覚的に独立させる

### 6. CTAは「色」と「言葉」で勝負
- ページ内で最もコントラストが高い色にする
- 背景が青系→CTAは赤/オレンジ、背景が白系→CTAは緑/赤
- CTA文言は動詞で終わる（「相談する」「今すぐ申し込む」「無料で試す」）
- CTAボタンの周囲にマイクロコピー（「30秒で完了」「無料」等）

### 7. スマホファーストで設計する
- 日本のLP閲覧は80%以上がスマホ
- 横幅375px〜を基準に設計し、PCは後から拡張する
- スマホでの指タップ領域は最低44px確保
- テキストは最低14px、メインコピーは最低28px

### 8. カラーリングの鉄則
- メインカラー1色 + アクセント1色 + 白/黒/グレーの構成
- 業種ごとの定番色を尊重する：
  - 法律・士業：ネイビー、ダークブルー、ゴールド
  - 医療・美容：白、ライトブルー、ピンク
  - EC・物販：ブランドカラー優先
  - 金融・保険：ブルー、グリーン（安心感）
  - 飲食・食品：赤、オレンジ、茶（食欲・温かみ）

### 9. 日本語タイポグラフィ
- ゴシック体（Noto Sans JP, BIZ UDGothic）：信頼・モダン・力強さ
- 明朝体（Noto Serif JP, Shippori Mincho）：高級感・伝統・感情
- 太めゴシック（M PLUS Rounded, Zen Maru Gothic）：親しみ・ポップ
- 欧文との混植時はベースラインを揃える
- Google Fontsで利用可能なフォントを使う

### 10. 縦書きテキストを効果的に使う
- 日本語は縦書きが自然に読める言語。FVの右端や左端に縦書きコピーを配置するだけで「日本っぽさ」が出る
- `writing-mode: vertical-rl` を使用
- 縦書きと横書きを**混在**させることでレイアウトに動きが出る
- 参考：音更電設、mamarich

### 11. 幾何学シェイプ/装飾要素を重ねる
- 写真の上に半透明のカラーシェイプ（丸、三角、不定形）を重ねる
- `clip-path` や `border-radius` でCSS実装
- 背景に淡いドット、斜線、波線パターンを敷く
- 小さなアイコンやイラストを散りばめる（position: absolute）

## Claudeが陥りやすいNG集（必読）

### NG1：海外SaaS風にしてしまう
- **症状**：余白たっぷり、Inter/Roboto、紫グラデーション、中央揃え1カラム
- **対策**：情報密度を上げる。要素を画面いっぱいに配置する。必ず日本語フォントを指定する

### NG2：フォントサイズが均一すぎる
- **症状**：メインコピー36px、サブコピー18px程度で差が小さい
- **対策**：メインコピーの最大文字は**最低60px**（モバイル）。サブコピーとの差は**3倍以上**

### NG3：装飾が足りない
- **症状**：プレーンテキストだけで文字装飾なし。蛍光マーカー・部分カラー・バッジなし
- **対策**：1つのコピー内で最低2種類の装飾を使う（色変え + サイズ変え、マーカー + 太字変え 等）

### NG4：写真の配置が小さい/弱い
- **症状**：200px程度の小さな写真、または写真なし
- **対策**：写真は画面面積の**40%以上**。人物写真は切り抜き（背景透過）か全面配置

### NG5：CTAが目立たない
- **症状**：CTAが背景と同系色、テキストリンクだけ
- **対策**：CTAは背景と**補色関係**の色。ボタンにシマーアニメーション。マイクロコピー付き

### NG6：全要素中央揃えの単調レイアウト
- **症状**：すべての要素がセンタリングされて上から順に並ぶだけ
- **対策**：左右非対称にする。写真を左/テキストを右、縦書きと横書きの混在、要素を重ねる

### NG7：コピーが説明的すぎる
- **症状**：「当社は〜を提供するサービスです」のような企業紹介文
- **対策**：ユーザーの悩みや感情に刺さる短いコピー。体言止め。問いかけ。数字

## アニメーション指針

### FVで使うべきアニメーション
1. **フェードイン + 上スライド**：基本中の基本。各要素を0.1〜0.2秒ずらして順番に表示
   ```css
   transform: translateY(20px); opacity: 0;
   → transform: translateY(0); opacity: 1;
   transition-delay: 0s, 0.15s, 0.3s ... /* 要素ごとにずらす */
   ```

2. **スケールイン**：人物写真の登場に。0.95→1.0のわずかな拡大
   ```css
   transform: scale(0.95); → transform: scale(1);
   ```

3. **CTAのシマーアニメーション**：ボタン上を光が走る
   ```css
   @keyframes shimmer {
     0%, 100% { left: -100%; }
     50% { left: 100%; }
   }
   ```

4. **数字のカウントアップ**：パターンFで使用。JSで0→目標値を0.8秒で

### やりすぎNG
- 3D回転、バウンス、派手なパーティクル（LPの信頼感を損なう）
- 表示に2秒以上かかるアニメーション（ユーザーが離脱する）
- スクロールジャック（FVでは禁止）

## 画像の取り扱い

### Unsplashでの検索キーワード例
| 業種 | 検索キーワード |
|------|--------------|
| 士業 | `japanese businessman suit`, `japanese lawyer office` |
| 医療 | `japanese doctor`, `japanese nurse patient` |
| 美容 | `japanese woman beauty`, `japanese skincare` |
| 飲食 | `japanese restaurant chef`, `japanese food` |
| EC | `japanese woman shopping`, `japanese lifestyle` |
| 採用 | `japanese worker`, `japanese team office` |
| 教育 | `japanese student`, `japanese classroom` |

### 写真のCSS処理パターン
1. **切り抜き配置**：`object-fit: cover` + `border-radius` で形を整える
2. **全面背景**：`position: absolute; inset: 0` + 半透明オーバーレイ
3. **写真にテキスト重ね**：写真の上にグラデーションオーバーレイ + 白文字
4. **幾何学マスク**：`clip-path: circle()` や `clip-path: polygon()` で形を変える

### 画像が用意できない場合
- **グラデーション背景 + テキスト**で代替（パターンD/Jのバリエーション）
- **アイコン・イラストSVG**を大きく使う
- **絶対にテキストだけのFVにしない**

## コーディングルール

### 技術スタック
- 基本：HTML + CSS + vanilla JS（ライブラリ不要のシンプル構成）
- 必要に応じて：React / Next.js / Tailwind CSS
- アニメーション：CSS animation優先、必要ならGSAP

### ファイル構成
```
index.html    ← メインHTML
style.css     ← スタイル
main.js       ← アニメーション・インタラクション
```

### CSS設計方針
- CSS変数でカラー・フォント・スペーシングを管理
- BEM風のクラス命名（`.hero__title`, `.hero__badge`）
- レスポンシブはモバイルファーストで `min-width` メディアクエリ

### 画像について
- 人物写真はUnsplashから適切なものを選ぶ
- `loading="eager"` + `fetchpriority="high"` でFVの画像を最優先読み込み
- WebP対応を意識するが、HTMLデモではjpg/pngでOK

## やってはいけないこと

### デザイン
- ミニマルすぎるデザイン（海外SaaS風の余白だらけ）
- Inter, Roboto, Arialなど欧文フォントだけの指定
- 紫グラデーション＋白背景のような「AIっぽい」デザイン
- 要素をすべて中央揃えにした単調なレイアウト
- 写真なしのテキストだけのファーストビュー
- コントラスト不足で読めないテキスト
- メインコピーが40px以下（モバイル基準）
- 装飾なしのプレーンテキストだけのコピー
- 角丸が統一されていない（8px, 12px, 20pxが混在等）

### コピー
- 「〜を提供いたします」「〜のソリューション」等のビジネス文体
- 主語が「当社は」「弊社は」で始まるコピー（ユーザー視点でない）
- 英語のキャッチコピー（日本のLP＝日本語で刺す）
- CTA文言が「送信」「申し込み」等の事務的な動詞

### コーディング
- `px`固定だけのフォントサイズ（`clamp()`を使うこと）
- メディアクエリなしのPC/モバイル共通サイズ
- `!important`の多用
- インラインスタイルの直書き
